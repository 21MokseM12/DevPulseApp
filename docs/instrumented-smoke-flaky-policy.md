# Instrumented Smoke Flaky Policy

## Цель

Разделить инфраструктурную нестабильность и функциональную регрессию для `Instrumented Smoke (Compose)` в CI.

## Критерии классификации падений

Падение считается **functional regression**, если:
- стабильно воспроизводится локально и в CI на одном и том же тест-кейсе;
- связано с нарушением ожидаемого UI flow или бизнес-логики.

Падение считается **flaky/infrastructure**, если:
- исчезает на автоматическом retry в том же CI job;
- сопровождается симптомами эмулятора/инфраструктуры (ADB disconnect, timeout загрузки emulator, UIAutomator/Compose idle timeout без повторяемого функционального сбоя).

## Политика retry и quarantine

- Для smoke job разрешен ровно 1 автоматический retry.
- Если после retry падение осталось — job считается red и блокирует merge.
- Quarantine допускается только для подтвержденного flaky-теста:
  1. завести issue с логами и скриншотами из `quality-instrumented-smoke`;
  2. временно исключить тест из CI набора;
  3. вернуть тест в основной blocking-набор не позже чем через 3 рабочих дня.

## SLA возврата

- Владелец flaky-issue назначается в день обнаружения.
- Статус пересматривается ежедневно до возврата теста в blocking smoke suite.
