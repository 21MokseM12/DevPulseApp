# Спецификация валидации credentials

## Область действия
- Политика применяется одинаково для входа и регистрации до сетевого запроса.
- Проверки выполняются по `trim()`-значению полей `login` и `password`.
- Приоритет: клиентская ошибка валидации имеет более высокий приоритет, чем серверная ошибка.

## Обязательные правила
- `login`: минимум `4` символа.
- `password`: минимум `8` символов.
- Допустимые символы для обоих полей: `[a-zA-Z0-9._-]`.
- `password` содержит минимум `1` букву и `1` цифру.

## Приоритет ошибок

### Логин
1. Пустое значение: `Введите логин.`
2. Длина меньше 4: `Логин должен содержать минимум 4 символа.`
3. Недопустимые символы: `Логин может содержать только латинские буквы, цифры и символы . _ -.`

### Пароль
1. Пустое значение: `Введите пароль.`
2. Длина меньше 8: `Пароль должен содержать минимум 8 символов.`
3. Недопустимые символы: `Пароль может содержать только латинские буквы, цифры и символы . _ -.`
4. Нет букв: `Пароль должен содержать хотя бы одну букву.`
5. Нет цифр: `Пароль должен содержать хотя бы одну цифру.`

## Примеры
- `valid login`: `user`, `user.name`, `user_name`, `user-name`
- `invalid login`: `abc`, `a bcd`, `юзер123`, `name!`
- `valid password`: `pass1234`, `Abcd0000`, `a1_b-c.d`
- `invalid password`: `12345678`, `abcdefgh`, `ab12`, `abc 1234`, `пароль12`, `pass12!@`
